# Spec

## Scope

Define the first concrete project task before implementation starts.

## Behavioral Discipline

- Record material assumptions, tradeoffs, and blocking questions here before moving to design or coding.
- Keep the first task narrow enough that its changes can be reviewed and verified directly.
- List non-goals so unrelated cleanup does not enter the first implementation slice.

## Acceptance Criteria

- The user goal is captured.
- Scope and non-goals are explicit.
- Open questions are either answered, non-blocking, or assigned to the user.
- Initial verification expectations are named.
